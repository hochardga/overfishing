## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** The first phase creates the shell, state model, styling system, and save plumbing that every later system depends on.
3. **Magic moment first.** The roadmap is structured so the fisher-to-operator role shift is achievable by the end of the core MVP work.
4. **Test as you go.** Reducers, selectors, and primary flows get verification during the phase that introduces them.
5. **Progressive enhancement.** Start with the smallest working version of each system, then layer density, polish, and consequence signals on top.
6. **Review before proceeding.** Push each completed phase as a PR and let a review agent such as [CodeRabbit](https://coderabbit.ai) review it before moving on.
7. **Protect the first-session arc.** If a task threatens pacing, clarity, or role change, cut scope before adding more features.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent
1. **Start a session:** Give your coding agent the phase prompt at the beginning of the current phase.
2. **Read selectively:** Each phase lists the exact PRD, vision, and GTM sections needed for that work.
3. **Let it work in order:** The agent should start from the first unchecked task and move downward without skipping.
4. **Aim to finish a phase:** Ideally complete a whole phase in one session. If not, resume from the next unchecked task later.
5. **Push a PR for review:** When a phase is complete, open a PR and let a review agent such as CodeRabbit inspect it before moving on.
6. **Update the status line:** Keep `**Status:**` and `**Current Phase:**` accurate as tasks are completed.

### Session Tips
- **Don't read everything.** Loading all documents at once wastes context and makes it easier to drift from the plan.
- **Don't skip tasks.** The order resolves dependencies intentionally.
- **Verify after each phase.** Run the app after completing a phase and confirm the demoable outcome is real.
- **Protect the arc.** If implementation reveals scope creep, preserve the first-session experience before adding more surface area.

### Prompt Templates
**Starting a new phase:**
> "Read docs/product-roadmap.md and find the current phase. Read only the Reference sections listed for that phase from docs/prd.md, docs/product-vision.md, and docs/gtm.md. Start on the first unchecked task. After completing each task, update the checkbox to [x] in the roadmap file."

**Resuming after a break:**
